# 📄 Динамічна Пагінація - Документація

## 🎯 Огляд

Нова система динамічної пагінації автоматично розподіляє контент документів на сторінки формату A4 з урахуванням реальної висоти елементів.

## ✨ Ключові особливості

### 🔄 **Динамічність**
- Автоматичне оновлення при зміні контенту
- Реагує на додавання/видалення тексту в реальному часі
- Підтримка різних типів документів (Bewerbung/Lebenslauf)

### 📏 **Точність**
- Вимірювання реальної висоти елементів (не символів)
- Врахування стилів та форматування
- Правильні пропорції A4 (210mm × 297mm)

### 🧠 **Інтелектуальність**
- Заголовки не відриваються від контенту
- Важливі секції зберігаються цілісними
- Адаптивний ліміт для різних типів документів

## 🔧 Технічна реалізація

### Основні функції:

#### `createDynamicPagination(htmlContent)`
Головна функція пагінації з аналізом символів:
```javascript
// Різні ліміти для різних документів
if (activeTab === "bewerbung") {
  CHARACTERS_PER_PAGE = 3500 // Ліміт для листів
} else {
  CHARACTERS_PER_PAGE = 2500 // Ліміт для резюме
}
```

#### `performIntelligentPagination(htmlContent, charactersPerPage, activeTab)`
Покращена функція з вимірюванням висоти:
```javascript
const MAX_PAGE_HEIGHT = 950 // пікселі для A4 сторінки
```

### Алгоритм роботи:

1. **Створення тимчасового контейнера** для вимірювання
2. **Парсинг HTML** та виділення стилів
3. **Вимірювання висоти** кожного елемента
4. **Розподіл по сторінках** з врахуванням:
   - Заголовків (не розривають)
   - Важливих секцій
   - Довгих елементів (таблиці, списки)
5. **Генерація HTML** для кожної сторінки

## 📊 Параметри налаштування

### Ліміти висоти:
- **MAX_PAGE_HEIGHT**: 950px (основний ліміт)
- **Заголовки**: 60% сторінки для перенесення
- **Секції**: 40% для критичних секцій
- **Малі елементи**: 30% для збереження на сторінці

### Ліміти символів:
- **Bewerbung**: 3500 символів/сторінку
- **Lebenslauf**: 2500 символів/сторінку
- **Адаптивний ліміт**: +20% для коротких документів

## 🧪 Тестування

### Файли для тестування:
1. **`test-pagination-manual.html`** - інтерактивний тест
2. **`test-pagination.html`** - автоматизований тест  
3. **`test-pagination.js`** - Node.js тест (потребує puppeteer)

### Як тестувати:

#### Основна сторінка:
1. Відкрийте http://localhost:8000
2. Перейдіть на вкладку "Lebenslauf" або "Anschreiben"
3. Заповніть поля форми
4. Спостерігайте автоматичну пагінацію в Live Preview

#### Інтерактивний тест:
1. Відкрийте `test-pagination-manual.html`
2. Використовуйте кнопки для додавання контенту
3. Спостерігайте зміну кількості сторінок
4. Перевіряйте статистику в реальному часі

### Команди для тестування:
```bash
# Запуск сервера
python3 -m http.server 8000

# Відкриття тестів
open test-pagination-manual.html
open http://localhost:8000
```

## 🎨 CSS стилі для A4

### Основні стилі сторінки:
```css
.document-page {
  width: 210mm;           /* A4 ширина */
  min-height: 297mm;      /* A4 висота */
  padding: 20mm;          /* Поля документа */
  background: white;
  box-shadow: 0 4px 20px rgba(0,0,0,0.15);
  page-break-after: always;
}
```

### Контент сторінки:
```css
.document-preview {
  font-size: 12pt;
  line-height: 1.6;
  color: #333;
  font-family: 'Times New Roman', serif;
}
```

## 🔍 Діагностика

### Debug функції:
- `window.debugTestPagination()` - тест обох вкладок
- `window.debugDataFlow()` - тест потоку даних
- `console.log()` в браузері для детального аналізу

### Типові проблеми:

1. **Не створюються сторінки**: Перевірте наявність контенту
2. **Неправильний розподіл**: Перевірте ліміти висоти
3. **Стилі не застосовуються**: Перевірте CSS селектори
